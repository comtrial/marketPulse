# ADR-004: MCP Server로 데이터 소스 노출

- **상태**: accepted
- **날짜**: 2026-03-27

## 맥락

구조화된 데이터(Neo4j 지식 그래프 + PostgreSQL 주문 데이터)를 LLM이 자율적으로 활용해야 한다.
LLM이 사용자 질문을 받으면 어떤 데이터를 조회할지 스스로 판단하고, 그 판단 과정이 추적 가능해야 한다.

## 결정

각 데이터 소스를 **MCP(Model Context Protocol) Server**로 래핑하여 LLM이 호출할 수 있는 도구로 노출한다.

- **Neo4j KG Server**: 인과 체인 쿼리, 속성 관계 탐색, 온톨로지 조회
- **주문 데이터 Server**: 속성별 트렌드, 국가별 히트맵, 채널별 비교 쿼리
- 모든 tool_call은 `tool_call_traces`에 로깅 → 왜 이 도구를 선택했는지 추적 가능

## 근거

| 방식 | LLM 자율성 | 확장성 | 추적성 |
|------|----------|-------|-------|
| 하드코딩된 쿼리 | 없음 | 새 쿼리마다 코드 수정 | 로그로 가능 |
| REST API + function calling | 가능 | 엔드포인트 추가 | 가능 |
| **MCP Server (선택)** | 가능 | 도구 단위 추가/제거 | tool_call_traces 네이티브 |

MCP는 LLM-도구 인터페이스의 표준 프로토콜로, 도구 추가/제거가 선언적이며 다른 MCP 클라이언트와도 호환된다.

## 결과

- **장점**: LLM이 질문 맥락에 따라 적절한 데이터 소스를 자율 선택
- **장점**: 도구 추가가 선언적 — 새 데이터 소스를 서버 하나로 추가 가능
- **장점**: tool_call_traces로 LLM의 추론 과정 완전 추적
- **단점**: MCP 서버 운영 오버헤드 (프로세스 관리, 헬스체크)
- **단점**: MCP 프로토콜 자체가 아직 초기 단계 — 스펙 변경 가능성
- **후속**: MCP 서버 구현 (Node.js or Python), tool 스키마 정의, traces 저장소 설계
