# ADR-002: Golden Example few-shot 기반 속성 추출

- **상태**: accepted
- **날짜**: 2026-03-27

## 맥락

뷰티 상품명에서 속성을 추출해야 하는데:
- "무기자차"가 자외선 차단 속성인지, "77"이 성분 함량인지 모델 번호인지, "약산성"이 pH 특성인지 마케팅 문구인지 — 모두 맥락 의존적
- 하위 카테고리마다 의미 있는 속성이 다름 (선크림의 "무기자차"는 토너에서 무의미)
- 룰 기반·키워드 기반으로는 해결 불가

## 결정

**Golden Example 기반 few-shot learning**으로 속성을 추출한다.

1. 입력 상품명을 벡터 임베딩
2. 벡터 검색으로 가장 유사한 top-N golden example 검색
3. 해당 golden example을 few-shot으로 LLM에 제공
4. LLM이 기존 속성 스키마에 맞춰 구조화된 속성 추출
5. 추출 결과는 Neo4j 개념 DB에 매핑 → 스키마 일관성 강제

## 근거

| 방식 | 맥락 이해 | 신규 속성 대응 | 일관성 |
|------|----------|-------------|-------|
| 룰 기반 (regex) | 불가 | 매번 룰 추가 필요 | 높으나 커버리지 낮음 |
| Fine-tuned 모델 | 가능 | 재학습 필요 | 학습 데이터 의존 |
| Zero-shot LLM | 가능 | 유연 | 스키마 드리프트 위험 |
| **Few-shot + Golden Example (선택)** | 가능 | 유사 예제로 가이드 | Neo4j 매핑으로 강제 |

few-shot은 fine-tuning 없이도 맥락 이해가 가능하고, golden example이 축적될수록 성능이 점진적으로 향상되는 선순환 구조를 만든다.

## 결과

- **장점**: golden example DB가 커질수록 few-shot 품질 향상 (선순환)
- **장점**: fine-tuning 없이 즉시 적용, 새 카테고리에도 유연 대응
- **장점**: Neo4j 매핑으로 집계용 스키마 일관성 보장
- **단점**: golden example 초기 구축 비용 (cold start)
- **단점**: 벡터 검색 품질이 few-shot 성능에 직접 영향
- **후속**: 임베딩 모델 선정, golden example 시드 데이터 구축, 벡터 DB 선정
