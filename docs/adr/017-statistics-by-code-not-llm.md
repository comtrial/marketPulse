# ADR-017: 통계 검증은 LLM이 아닌 코드가 수행

- **상태**: accepted
- **날짜**: 2026-03-30

## 맥락

PatternScout가 속성 간 통계적 관계(lift, correlation, p-value)를 계산해야 한다. LLM이 계산할지, 코드가 계산할지 결정해야 한다.

## 결정

**코드(SQL 집계 + scipy.stats)가 계산하고, LLM은 "어떤 속성 쌍을 검사할지"만 판단한다.**

## 근거

LLM이 "lift가 1.54입니다"라고 생성하면:
- 이 숫자가 SQL 집계 결과인지, 학습 데이터에서 온 근사치인지, 환각인지 구분 불가
- 재현 불가능 — 같은 질문에 다른 숫자가 나올 수 있음
- evidence로 저장해도 검증 방법이 없음

코드가 계산하면:
- `COUNT(*) FILTER WHERE ... / total`의 결과이므로 재현 가능
- evidence에 SQL 결과를 그대로 저장 → 누구든 쿼리로 재현·검증 가능
- 같은 데이터에 같은 결과 보장 (결정적)

LLM의 역할 한정: `compute_cooccurrence_lift(attr_a="비건", attr_b="무기자차", ...)` 형태로 파라미터만 선택. "비건-무기자차 쌍이 의미 있을 것 같다"는 가설 선택이 LLM의 가치.

## 임계치 근거

| 임계치 | 값 | 근거 |
|--------|-----|------|
| lift > 1.3 | 약한 연관의 하한 | 역학 연구에서 사용. 1.0=독립, 1.3=우연 대비 30% 이상 |
| sample_size > 30 | 최소 표본 | 중심극한정리에 의한 정규 근사 성립 하한 |
| \|correlation\| > 0.6 | 강한 상관 하한 | 사회과학 통계 관례 |
| p < 0.05 | 통계적 유의성 | 표준 유의수준 |

이 임계치는 MVP 기본값이며, 도메인 전문가와의 튜닝을 통해 조정해야 한다.

## 결과

- **장점**: 숫자의 신뢰성 보장, 재현 가능, 검증 가능
- **장점**: evidence 필드에 저장된 수치를 DB에서 직접 재현·검증 가능
- **단점**: LLM이 "이 lift가 높은 건가?"를 판단 못함 → 임계치를 코드로 정의해야 함
