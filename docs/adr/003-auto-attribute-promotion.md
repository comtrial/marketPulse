# ADR-003: 자동 속성 승격 루프

- **상태**: accepted
- **날짜**: 2026-03-27

## 맥락

뷰티 업계는 신규 트렌드 속성("마이크로바이옴", "시카", "병풀")이 계속 등장한다.
속성 사전을 하드코딩하면 시장 변화를 따라갈 수 없고, 매번 사람이 수동으로 추가하면 병목이 된다.

## 결정

**자동 감지 + 자동 승격** 루프를 구현한다.

```
1. LLM 속성 추출 시 정규 스키마에 없는 속성 → additionalAttrs에 수집
2. 배치 집계로 additionalAttrs 키의 등장 빈도 카운팅
3. 임계치(설정값) 초과 → 자동 승격
   - Neo4j에 새 Attribute 노드 생성
   - Tool 스키마에 필드 추가
   - Golden Example DB에 해당 속성 포함 예제 추가
4. 이후 추출에서 해당 속성이 정규 필드로 자동 추출
5. 새 golden example이 임베딩 DB에 반영 → few-shot 성능 향상
```

## 근거

- 완전 수동: 병목, 트렌드 대응 느림
- 완전 자동 (무조건 승격): 노이즈 속성 폭증
- **빈도 기반 자동 승격 (선택)**: 시장에서 실제로 반복 등장하는 속성만 승격, 노이즈 필터링

## 결과

- **장점**: 사람 개입 없이 시장 트렌드 자동 반영
- **장점**: 승격된 속성이 golden example에 반영 → few-shot 성능 점진적 향상 (셀프 퍼페추에이팅 루프)
- **장점**: 임계치 조정으로 승격 민감도 제어 가능
- **단점**: 임계치 튜닝 필요 (너무 낮으면 노이즈, 너무 높으면 지연)
- **단점**: 동음이의어 속성의 자동 분리가 어려울 수 있음
- **후속**: 임계치 초기값 설정, additionalAttrs 집계 배치 구현, 승격 히스토리 로깅
