# ADR-018: PatternScout의 구조화된 탐색 절차

- **상태**: accepted
- **날짜**: 2026-03-30

## 맥락

PatternScout에게 "자유롭게 가설을 세우고 검증하세요"라고 하면, LLM은 위험 회피적으로 행동하여 안전한 읽기 도구만 반복 호출하고 propose_relationship은 호출하지 않는다.

## 결정

**명시적 5단계 탐색 절차를 system prompt에 제공한다.** LLM은 절차를 따르되, "어떤 속성 쌍을 검사할지"는 LLM이 판단.

## 근거

자유 탐색 vs 구조화 탐색 비교:

| 접근 | 장점 | 단점 |
|------|------|------|
| 자유 탐색 | 유연, 예상 못한 패턴 발견 가능 | LLM이 뭘 할지 몰라 안전한 도구만 호출 |
| 구조화 탐색 | 예측 가능, 반드시 제안까지 도달 | 절차 밖 패턴을 놓칠 수 있음 |

**절충**: 구조화 5단계를 기본으로 하되, "위 절차 외에 추가 패턴이 보이면 test_hypothesis로 검증 후 제안하세요"를 열어둠.

## 5단계 절차

1. 직전 분석의 주요 속성 파악
2. 같은 국가+카테고리의 상위 5개 속성 조회 (후보 선정)
3. 동시 출현 lift 계산 → lift > 1.3 이면 SYNERGY 제안
4. 시계열 상관 계산 → |correlation| > 0.6 이면 TEMPORAL_CORRELATION 제안
5. 다른 국가 비율 비교 → p < 0.05 이면 CROSS_MARKET_GAP 제안

## 결과

- **장점**: LLM이 반드시 propose_relationship까지 도달
- **장점**: 임계치가 명시적이므로 Eval에서 "왜 이 관계를 제안했는지" 검증 가능
- **장점**: Step별 trace가 5단계 절차와 매핑되어 디버깅 용이
- **단점**: 절차가 경직적 — 5단계에 해당하지 않는 패턴은 열린 탐색(test_hypothesis)에 의존
