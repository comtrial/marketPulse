# MarketPulse — 프로젝트 설명

## 프로젝트 요약

CBE(Cross-Border Ecommerce) 도메인의 실 주문 데이터에서 뷰티/화장품 상품의 비정형 속성을 구조화하고, 국가별 시장 인사이트를 자율 생성하는 LLM 멀티에이전트 시스템이다. 룰 기반으로는 해결할 수 없는 상품명 속성 추출 문제를 정의하고, Tool Use + Dynamic Few-Shot 추출 파이프라인, Neo4j 인과 체인 위에서 LLM이 데이터 소스를 자율 조합하는 ReAct 오케스트레이터, 통계적 패턴 탐지로 온톨로지를 지속 확장하는 PatternScout 에이전트를 설계·구현했다. 시스템이 사용될수록 지식 기반이 성장하는 구조.

---

### | 속성 추출을 위한 Tool Use + Dynamic Few-Shot 파이프라인

주문 데이터에서 국가별·채널별 인사이트를 도출하려면, 비정형 상품명에서 구조화된 속성을 먼저 추출해야 한다. 뷰티 상품명에서 "무기자차"는 선크림에선 UV 차단 방식이지만 토너에선 무의미하고, "77"이 성분 함량인지 모델 번호인지는 카테고리에 따라 달라진다. 이 맥락 의존성 때문에 룰 기반 분류는 원천적으로 불가능하다. LLM 단독으로는 대량 추출 시 출력 형식이 일관되지 않아 후속 집계가 불가능하므로, Tool Use로 출력 구조를 강제하고 Dynamic Few-Shot으로 값 표기 형식을 유도하는 2-레이어 전략을 설계했다. Few-Shot 예시는 검수된 데이터를 ChromaDB에 임베딩해두고, 입력 상품명과의 벡터 유사도 검색으로 동적 선택하여 LLM에 주입한다. LangChain 대신 Anthropic API의 Tool Use와 ReAct 패턴으로 직접 구현하여 도구 호출 로깅, 프롬프트 버전 관리, 검증 파이프라인을 완전히 제어할 수 있도록 했다.

### | 인과 추론과 통계 집계를 LLM이 자율적으로 조합하여 인사이트를 도출하는 에이전트 설계

SQL 집계로는 "비건 58%"라는 숫자만 나오고, "왜 58%인가?"에는 답할 수 없다. Neo4j에 기후→피부고민→기능→성분의 인과 체인을 온톨로지로 구축하고, LLM이 while-loop ReAct 패턴으로 Neo4j(인과)와 PostgreSQL(집계)을 자율 조합하여 인사이트를 생성한다. 도구 정의는 Pydantic 모델 기반으로 JSON 스키마가 코드에서 자동 파생되며, 모든 도구 호출의 reasoning + decision + result를 trace에 기록하여 LLM의 판단 과정을 추적할 수 있다.

### | PatternScout: 데이터에서 통계적 패턴을 탐지하여 온톨로지를 지속 확장

사전 정의된 인과 체인만으로는 데이터에서 새로 발생하는 속성 간 관계를 포착할 수 없다. 주문 데이터에서 통계적 공존 패턴을 자동 탐지하는 별도 에이전트를 설계하여, 시스템이 사용될수록 온톨로지가 확장되는 구조를 만들었다. LLM에게는 "어떤 속성 쌍을 검사할지"의 탐색 판단만 맡기고, 동시 출현 빈도나 시계열 상관계수 같은 통계 수치는 SQL 집계와 통계 라이브러리 전용 도구로만 산출한다. 탐지된 패턴은 사용자 검토와 승인을 거쳐 온톨로지에 편입되며, 이후 동일 질문에 대해 AnalystAgent가 승인된 관계의 통계적 근거를 활용하여 답변하는 것을 Before/After 비교로 정량 확인했다.

### | 시스템 변경이 답변 품질에 미치는 영향을 추적하는 Eval 설계

관계 승인, 프롬프트 수정, 데이터 추가 등의 변경이 최종 답변 품질을 실제로 개선했는지 정량 판단할 수 있는 Eval을 설계했다. 패턴 탐지 현황, 답변 내 승인 관계 활용률, 인과 근거 커버리지, 시스템 효율을 4축으로 측정한다. 온톨로지 확장에 따른 품질 개선과 PatternScout 실행 비용 증가의 트레이드오프를 세션별 비용 관리로 추적하여, 개선이 비용 증가를 정당화하는지를 데이터로 판단할 수 있다.

---

### 주요 구현 내용

- 비정형 상품명에서 맥락 의존적 속성을 구조화하는 Few-Shot + Tool Use 추출 파이프라인 구현
- LLM이 Neo4j(인과) + PostgreSQL(집계) 도구를 자율 조합하는 ReAct 오케스트레이터 구현
- Pydantic 기반 도구 스키마 자동 생성 구조 설계 (도구 추가 시 단일 파일 수정)
- 통계적 패턴을 코드가 계산하고 LLM이 제안하는 PatternScout 에이전트 구현
- 제안→사용자 승인→온톨로지 편입→답변 변화의 Human-in-the-loop 지식 확장 파이프라인 구현
- Eval 평가 지표 모니터링, 변경 전후 스냅샷 비교 시스템 구현
- 의사결정 과정을 Think/Act/Observe로 시각화하는 단일 페이지 대시보드 구현

### 기술 스택

Python, FastAPI, Claude API (Sonnet 4 / Tool Use), Neo4j, PostgreSQL, ChromaDB, scipy, Next.js, React, shadcn/ui, Recharts
