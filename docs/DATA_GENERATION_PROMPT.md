# MarketPulse 주문 데이터 생성 명세서

> **이 문서의 목적**: 다른 AI 에이전트가 이 문서를 읽고, MarketPulse 시스템의 주문 데이터 1,000건을 정확하게 생성할 수 있도록 하는 완전한 명세. 이 데이터는 실제 CJ대한통운 CBE 시스템에서 각 플랫폼의 배치 수집기가 적재하는 주문 데이터와 동일한 구조를 가진다.

> **핵심 원칙**: 이 데이터에는 **의도적인 패턴**이 심어져 있어야 한다. 시스템이 이 패턴을 정확히 감지하는지를 검증하는 것이 목적이다. 따라서 데이터 생성자(에이전트)는 반드시 아래 명세를 정확히 따라야 하며, 임의로 패턴을 변형하면 안 된다.

---

## 1. 출력 형식

### 파일 — 플랫폼별 3개 파일로 분리

```
orders_cafe24.csv   — Cafe24 플랫폼 주문
orders_qoo10.csv    — Qoo10 플랫폼 주문
orders_shopee.csv   — Shopee 플랫폼 주문
```

실제 프로덕션에서 각 플랫폼 배치 수집기가 별도 테이블에 적재하는 구조와 동일.
3개 파일 모두 동일한 CSV 스키마를 따른다.

### CSV 스키마 (컬럼 순서 고정)

```csv
order_id,order_date,platform,destination_country,brand,product_name,product_type,quantity,unit_price_usd
```

### 컬럼 정의

| 컬럼 | 타입 | 설명 | 예시 |
|------|------|------|------|
| `order_id` | string | 고유 주문 ID. 플랫폼별 접두어: `CF24-{6자리}`, `QO10-{6자리}`, `SHPE-{6자리}` | `CF24-000001` |
| `order_date` | string | 주문일. `YYYY-MM-DD` 형식. 해당 월 내 랜덤 날짜 | `2025-10-15` |
| `platform` | string | 플랫폼. 아래 3개 중 하나 | `cafe24` |
| `destination_country` | string | 배송 국가. 아래 3개 중 하나 | `JP` |
| `brand` | string | 브랜드명. 아래 4개 중 하나 | `Innisfree` |
| `product_name` | string | **상품명 전체 텍스트** (속성 추출의 입력값) | `이니스프리 데일리 UV 프로텍션 크림 SPF50+ PA++++ 50ml 톤업 비건` |
| `product_type` | string | 상품 소분류. 아래 목록 중 하나 | `sunscreen` |
| `quantity` | integer | 주문 수량. 1~5 사이 랜덤 | `2` |
| `unit_price_usd` | float | 단가 (USD). 소수점 2자리 | `18.50` |

## 2. 고정 파라미터

### 플랫폼 (3개)
```
cafe24   — 한국 자사몰 기반. KR, JP, SG 모두 커버
qoo10    — 일본/싱가포르 특화. JP, SG만
shopee   — 동남아 특화. SG만
```

### 국가 (3개)
```
KR — 한국 (내수)
JP — 일본
SG — 싱가포르 (동남아 대표)
```

### 플랫폼 × 국가 매핑 (이 조합만 허용)
```
cafe24 + KR  ✅
cafe24 + JP  ✅
cafe24 + SG  ✅
qoo10  + JP  ✅
qoo10  + SG  ✅
shopee + SG  ✅

허용되지 않는 조합:
qoo10  + KR  ❌
shopee + KR  ❌
shopee + JP  ❌
```

### 브랜드 (4개)
```
Innisfree  — 이니스프리. 아모레퍼시픽 계열. 자연주의 컨셉.
Torriden   — 토리든. 저분자 히알루론산 특화 브랜드.
Roundlab   — 라운드랩. 독도 토너로 유명. 심플 스킨케어.
Skinfood   — 스킨푸드. 식물/식품 원료 기반.
```

### 상품 유형 (5개)
```
sunscreen  — 선크림/선케어
toner      — 토너/스킨
serum      — 세럼/에센스/앰플
cream      — 크림/모이스처라이저
lip        — 립틴트/립밤/립마스크
```

### 기간
```
2025-10, 2025-11, 2025-12, 2026-01, 2026-02, 2026-03
(6개월, 월 단위)
```

## 3. 데이터 분포

### 3.1 월별 총 건수 (1,000건 분배)

```
2025-10: 150건
2025-11: 160건
2025-12: 170건
2026-01: 170건
2026-02: 175건
2026-03: 175건
합계:   1,000건
```

### 3.2 국가별 분배 (각 월 내에서)

```
각 월의 건수를 아래 비율로 분배:
  KR: 30%
  JP: 40%
  SG: 30%

예시: 2025-10월 150건이면
  KR: 45건
  JP: 60건
  SG: 45건
```

### 3.3 플랫폼 배분 (국가 내에서)

```
KR 주문:
  cafe24: 100%

JP 주문:
  cafe24: 40%
  qoo10:  60%

SG 주문:
  cafe24: 20%
  qoo10:  30%
  shopee: 50%
```

### 3.4 상품 유형 배분 (국가 내에서)

```
KR:
  sunscreen: 20%, toner: 25%, serum: 25%, cream: 20%, lip: 10%

JP:
  sunscreen: 35%, toner: 20%, serum: 20%, cream: 15%, lip: 10%

SG:
  sunscreen: 30%, toner: 25%, serum: 20%, cream: 15%, lip: 10%
```

### 3.5 브랜드 배분 (균등)

```
각 국가 × 상품유형 조합 내에서 4개 브랜드를 대략 균등하게 배분.
정확히 25%씩이 아니어도 되지만, 한 브랜드가 50% 이상 차지하면 안 됨.
```

## 4. 의도적 패턴 (핵심 — 반드시 정확히 반영)

### 패턴 A: 일본에서 "비건" 속성 상승 트렌드

```
대상: destination_country = "JP", product_type = "sunscreen"
패턴: 상품명에 "비건" 키워드가 포함된 주문의 비율이 월별로 상승

월별 비율:
  2025-10: 18%  (JP sunscreen 주문 중 비건 포함 비율)
  2025-11: 25%
  2025-12: 34%
  2026-01: 42%
  2026-02: 51%
  2026-03: 58%

구현 방법:
  JP sunscreen 주문을 생성할 때, 위 비율에 맞춰
  상품명에 "비건"을 포함/미포함 결정.
  
  예: 2026-03 JP sunscreen이 24건이면, 14건(58%)은 "비건" 포함, 10건은 미포함.
```

### 패턴 B: 싱가포르에서 "워터프루프" 일관 우세

```
대상: destination_country = "SG", product_type = "sunscreen"
패턴: 상품명에 "워터프루프" 키워드가 포함된 비율이 모든 월에서 70~80%

월별 비율:
  모든 월: 72~78% 범위 내 (약간의 자연스러운 변동)

구현 방법:
  SG sunscreen 주문의 약 75%에 "워터프루프" 포함.
  월마다 정확히 같지 않고 ±3% 정도 자연스럽게 변동.
```

### 패턴 C: 일본에서 "톤업" 하락 트렌드

```
대상: destination_country = "JP", product_type = "sunscreen"
패턴: "톤업" 키워드 포함 비율이 월별로 하락

월별 비율:
  2025-10: 67%
  2025-11: 62%
  2025-12: 56%
  2026-01: 50%
  2026-02: 46%
  2026-03: 43%
```

### 패턴 D: 블루오션 조합 — JP sunscreen

```
대상: destination_country = "JP", product_type = "sunscreen"
패턴: 아래 속성 조합별로 의도적인 주문량/상품 다양성 차이

조합 1 — 레드오션:
  "톤업" 단독 (비건 아님): 높은 주문량, 많은 상품 종류
  → 모든 브랜드에서 톤업 선크림이 있어야 함

조합 2 — 블루오션:
  "비건" + "무기자차" 조합: 중간 주문량, 매우 적은 상품 종류
  → Innisfree 1개 상품에서만 이 조합 등장
  → 하지만 그 1개 상품의 주문량이 꽤 높음 (반복 구매)

조합 3 — 기회:
  "비건" + "톤업" 조합: 중간 주문량, 소수 상품
  → 2~3개 상품에서 등장
```

### 패턴 E: 신규 속성 "마이크로바이옴" 등장

```
대상: 모든 국가, product_type = "toner" 또는 "serum"
패턴: 2026-01부터 상품명에 "마이크로바이옴" 키워드 등장, 점진적 증가

월별 건수 (전체, 국가 무관):
  2025-10: 0건
  2025-11: 0건
  2025-12: 0건
  2026-01: 3건
  2026-02: 8건
  2026-03: 25건  ← 임계치(20) 돌파. 승격 후보 감지 대상.

브랜드: Torriden에서만 등장.
상품명 예시: "토리든 다이브인 마이크로바이옴 토너 200ml 유산균 피부장벽"
```

### 패턴 F: 국가별 성분 선호 차이

```
대상: 모든 국가, product_type = "toner" 또는 "serum"

JP에서 인기 성분:
  "히알루론산" 포함 비율: 40%
  "센텔라/시카" 포함 비율: 30%
  "나이아신아마이드" 포함 비율: 25%

SG에서 인기 성분:
  "나이아신아마이드" 포함 비율: 45%
  "히알루론산" 포함 비율: 35%
  "센텔라/시카" 포함 비율: 15%

KR에서 인기 성분:
  "센텔라/시카" 포함 비율: 35%
  "히알루론산" 포함 비율: 30%
  "나이아신아마이드" 포함 비율: 25%

이 비율은 6개월 내내 대략 유지 (시계열 변화 없음, 국가별 차이만).
```

### 5.3 속성 키워드 풀 (상품명 뒤에 붙는 것들)

```
[기능 속성 — functional_claims]
톤업, 노세범, 수분, 보습, 진정, 미백, 브라이트닝, 각질케어,
워터프루프, 자외선차단, 피부장벽, 항산화, 수분폭탄, 수분진정

[가치 속성 — value_claims]
비건, 저자극, 무향, 약산성, 산호초안전, 무기자차, 대용량

[피부 타입]
민감성, 지성, 건성, 복합성

[성분 키워드 — 상품명에 자연스럽게 포함]
히알루론산, 센텔라, 시카, 나이아신아마이드, 녹차, 세라마이드,
어성초, 프로폴리스, 알로에, 마데카소사이드,
유산균, 마이크로바이옴(패턴E용, 2026-01부터만)

[선크림 전용]
SPF50+, PA++++, SPF30, PA+++

[립 전용]

### 5.4 상품명 생성 시 규칙

```
1. 속성 키워드는 상품명 뒤에 공백으로 구분하여 자연스럽게 붙인다.
   예: "이니스프리 데일리 UV 프로텍션 크림 SPF50+ PA++++ 50ml 톤업 비건 무기자차"

2. 모든 선크림에는 반드시 "SPF{숫자}" 와 "PA{+기호}"가 포함되어야 한다.
   대부분 SPF50+ PA++++. 간혹 SPF30 PA+++ (10% 이하).

3. 같은 제품라인이어도, 속성 키워드 조합이 다르면 다른 상품이다.
   "이니스프리 UV 크림 SPF50+ 톤업" ≠ "이니스프리 UV 크림 SPF50+ 톤업 비건"
   → 이 차이가 속성 추출과 집계의 핵심.

4. 립 제품에는 SPF, PA, 워터프루프, 비건 같은 속성을 붙이지 않는다.
   (립밤에 "비건"을 붙이는 건 가능하지만 소수.)
   대신 컬러번호, 컬러명, 질감(매트/글로시)이 핵심 속성.

5. 토너/세럼/크림에는 SPF, PA를 붙이지 않는다.
   대신 성분 키워드(히알루론산, 센텔라 등)와 기능(수분, 진정 등)이 핵심.

6. 같은 브랜드+제품라인의 상품이 반복 주문될 수 있다 (인기 상품 = 반복 구매).
   단, 1개 상품이 100건 이상 차지하면 안 됨 (비현실적).
   인기 상품은 15~30건 정도의 반복이 자연스러움.

7. 상품명의 길이는 최소 15자, 최대 80자 범위.
```

## 7. 패턴 검증 체크리스트

데이터 생성 후, 아래 체크리스트를 통해 패턴이 올바르게 반영되었는지 검증해야 한다.

```
□ 총 건수: 정확히 1,000건
□ 월별 건수: 150, 160, 170, 170, 175, 175
□ 허용되지 않는 플랫폼×국가 조합 없음 (qoo10+KR, shopee+KR, shopee+JP)
□ 모든 상품명이 15자 이상
□ 모든 선크림에 SPF 표기 있음
□ 립 제품에 SPF 표기 없음

패턴 A 검증:
□ JP sunscreen 중 "비건" 포함 비율:
  10월 ≈18%, 11월 ≈25%, 12월 ≈34%, 1월 ≈42%, 2월 ≈51%, 3월 ≈58%
  (±3%p 허용)

패턴 B 검증:
□ SG sunscreen 중 "워터프루프" 포함 비율: 모든 월 72~78%

패턴 C 검증:
□ JP sunscreen 중 "톤업" 포함 비율:
  10월 ≈67%, 11월 ≈62%, 12월 ≈56%, 1월 ≈50%, 2월 ≈46%, 3월 ≈43%
  (±3%p 허용)

패턴 D 검증:

  (= 상품 다양성 낮음, 하지만 주문 건수는 여러 건 — 반복 구매)

패턴 E 검증:
□ "마이크로바이옴" 포함 주문: 10월~12월 0건, 1월 3건, 2월 8건, 3월 25건

□ JP toner+serum 중 "히알루론산" 포함 ≈40%
□ SG toner+serum 중 "나이아신아마이드" 포함 ≈45%
□ KR toner+serum 중 "센텔라" 또는 "시카" 포함 ≈35%

## 9. 주의사항

```
1. 상품명은 반드시 한국어로 작성한다.
   브랜드명만 영어/한글 혼용 가능 (CSV의 brand 컬럼은 영어, 상품명 내 브랜드는 한글).

2. product_type은 반드시 영어 소문자로 (sunscreen, toner, serum, cream, lip).
   상품명 내에서는 한글로 (선크림, 토너, 세럼, 크림, 틴트 등).

3. 패턴 A~F의 비율은 ±3%p 오차를 허용하지만, 트렌드의 방향성은 반드시 유지.
   예: 패턴 A에서 "비건" 비율이 10월에 20%이고 11월에 19%로 떨어지면 ❌.
   반드시 단조 증가해야 함.

4. 같은 상품명이 여러 주문에서 반복될 수 있다 (인기 상품).
   하지만 전체 1,000건에서 고유 상품명은 최소 80종 이상이어야 한다.

5. order_date는 해당 월 내에서 1일~28일 사이의 랜덤 날짜.
   29~31일은 월마다 일수가 다르므로 사용하지 않는다.

6. quantity는 대부분 1~2, 간혹 3~5. 평균 1.5 정도.

7. 속성 키워드 간 조합 시 모순이 없어야 한다.
   예: "무기자차" + "유기자차"가 동시에 붙으면 ❌.
   예: "저자극" + "레티놀" 조합은 가능 (비현실적이지만 상품명에 나올 수는 있음).

8. 데이터 생성 후 반드시 섹션 7의 검증 체크리스트를 수행하고,
   통과하지 못한 항목이 있으면 데이터를 수정하여 재생성할 것.
```
